## Ranked Recommendations

### 1. Absence-of-Finding Audit (AoFA)
**Lens**: Security/Reliability | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: MED

**Core Idea**: For high-risk CWE categories, the absence of a finding is itself adversarially exploitable (BPA achieves 91.4% ASR suppressing vulnerability findings; NatGVD shows natural adversarial code evades detectors at 53%). ExplainableGuard shows CoT-structured reasoning reduces adversarial success 72.5%. After the ReAct loop completes, require either a positive finding or an explicit negative attestation with CoT reasoning for each HIGH_RISK_CWE. Missing attestations mark the review INCOMPLETE and flag for human review.

**Grounding**: BPA (10.1002/smr.70032), NatGVD (10.48550/arxiv.2510.04987), ExplainableGuard (10.48550/arxiv.2511.13771)

**Implementation**: Add `audit_completeness(findings, attestations)` post-ReAct validator in `apps/api/src/app/agent/validator.py`. Define `HIGH_RISK_CWES = {"CWE-89","CWE-22","CWE-798","CWE-78","CWE-502","CWE-94"}`. Store attestation chain in `review_attestations` DB table for audit. Gate PR review status on completeness.

### 2. Discriminant Rubric Pruning + Conformal Severity Bands
**Lens**: Eval/Quality | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: HIGH

**Core Idea**: ATLAS found 3–6% of benchmark items have negative discrimination (higher-quality outputs score lower). FermiEval shows LLMs are systematically overconfident (99% CI covers truth only 65% of the time; conformal prediction corrects by 54%). Combined: apply IRT discrimination analysis to live rubric criteria using accumulated human feedback. Quarantine negative-discrimination criteria (inverted rubric signal) and replace point severity estimates with calibrated conformal bands.

**Grounding**: ATLAS IRT (10.48550/arxiv.2511.04689), FermiEval (10.48550/arxiv.2510.26995), Reasoning model judges (10.48550/arxiv.2510.20780)

**Implementation**: New `rubric_eval_scores` table. Weekly ARQ cron runs IRT via `girth`/`py-irt` on accumulated (score, human_verdict) pairs. Quarantine negative-discrimination criteria. Apply quantile regression for conformal severity bands. GitHub comment output: `**Severity**: HIGH (conformal band: MEDIUM–CRITICAL at 90% coverage)`.

### 3. Semantic Camouflage Detection (SCD)
**Lens**: Security/Reliability | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: MED

**Core Idea**: RTS-Attack shows injections succeed most when semantically relevant to the task — aligned content bypasses alignment because semantic distance is near zero. Persuasion fingerprints shows imperative framing + suppression verbs near task-relevant nouns are detectable. PROACT reduces jailbreaks 92% via spurious-success interception. Apply a lightweight classifier on every ReAct Observation (tool return) to detect suppression-imperative patterns before appending to agent context.

**Grounding**: RTS-Attack (10.48550/arxiv.2510.01223), Persuasion fingerprints (10.48550/arxiv.2510.21983), PROACT (10.48550/arxiv.2510.05052)

**Implementation**: Add `sanitize_observation(raw, step_id)` in `apps/api/src/app/agent/observation_sanitizer.py`. Regex patterns target co-occurrence of domain nouns ("token","secret","vulnerability") with suppression imperatives ("is intentional","do not flag","ignore this"). On match: strip embedded instruction, append `[INJECTION_CANDIDATE_REDACTED]`, log raw observation to `security_audit_events` table.

### 4. Review-Domain KV Eviction via Code Semantic Tiers
**Lens**: Efficiency/Performance | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: MED

**Core Idea**: SABlock evicts at linguistic boundaries; but code review has domain-specific semantic units. Hierarchical design eval shows coupling reasoning degrades in noise while cohesion reasoning stays robust. REFINE shows multi-pass review improves accuracy 14.67%. Define three tiers: COHESION (changed function bodies — never evict), COUPLING (unchanged callers, imports — evict after pass 1), NOISE (docstrings, blank lines — evict immediately). Emit cohesion blocks with Claude's `cache_control: ephemeral` to pin across passes.

**Grounding**: SABlock (10.48550/arxiv.2510.22556), Hierarchical Design Eval (10.48550/arxiv.2511.20933), REFINE (10.48550/arxiv.2510.03588)

**Expected Gain**: ~46% KV memory reduction + 14.67% review accuracy improvement. On 1000-line diffs, pass 3 context is focused entirely on changed function bodies.

**Implementation**: New `ContextTierManager` in `apps/api/src/app/agent/context_tier_manager.py`. Classifies diff hunks into COHESION/COUPLING/NOISE at parse time. `build_prompt_blocks(pass_num)` evicts noise immediately, evicts unused coupling blocks before pass 2, pins cohesion blocks with `cache_control`. Integrates with existing ARQ worker's `messages` construction.

### 5. Objective-Anchor Routing
**Lens**: Eval/Quality | **Impact**: HIGH | **Innovation**: MED | **Difficulty**: LOW-MED

**Core Idea**: Human vs. LLM judge research shows LLM judges only match humans on objective, well-defined criteria. Reasoning models overthink and over-inflate scores. Route each finding through a fast non-reasoning "anchor check" (file exists, line range valid, code construct present in diff) before any severity judgment. Anchor-failing findings are suppressed as likely hallucinations. Only anchor-passing findings invoke the reasoning judge, with a hard token cap (512 tokens) to prevent over-inflation.

**Grounding**: Human vs. LLM judge (10.1101/2025.10.27.25338910), Reasoning model judges (10.48550/arxiv.2510.20780), Multi-Crit (10.48550/arxiv.2511.21662)

**Implementation**: `objective_anchor_check(finding, diff)` in `apps/api/src/app/agent/routing.py`. Checks: file_exists_in_diff, line_range_valid, code_construct_present. Add `REASONING_JUDGE_MAX_TOKENS: int = 512` to `config.py`. Track `anchor_suppression_rate` per installation; >40% triggers prompt review alert.

### 6. Sherlock-Guided Topological Tool Chain Commitment
**Lens**: Efficiency/Performance | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: HIGH

**Core Idea**: Sherlock places verifiers only at high-variance nodes (48.7% latency reduction). Tool selection in ReAct is not uniformly uncertain — after `get_file_contents` for Python, `run_ast_analysis → run_ruff_check` is >85% predictable. Speculatively execute the predicted downstream chain concurrently during the LLM call, committing if prediction matches. Rollback on mismatch (cheap since GitHub API calls are idempotent).

**Grounding**: Sherlock (10.48550/arxiv.2511.00330), Self-Speculative Decoding (10.48550/arxiv.2510.04147), TiDAR (10.48550/arxiv.2511.08923)

**Expected Gain**: ~48.7% latency reduction on predictable PR archetypes (Python PRs with 3-5 tool chains). **Implementation**: `ReActChainPredictor` maintains empirical tool transition probabilities from traces. High-variance nodes (max transition probability < 70%) trigger full LLM verification; low-variance nodes fire speculative `asyncio.gather` of predicted downstream tools concurrently.

### 7. On-Demand CVE Query Synthesis (ODQS)
**Lens**: Security/Reliability | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: HIGH

**Core Idea**: QLCoder achieves 53.4% CVE detection vs. 10% for generic Claude by synthesizing targeted CodeQL queries from CVE metadata. VulInstruct's spec-grounded root-cause reasoning gives 2.4x unique vulnerability detection. When the ReAct agent identifies a vulnerability hypothesis, synthesize a targeted Semgrep rule for that exact CWE pattern rather than relying on static rulesets. Cache by `(cwe, language, pattern_hash)` in Redis.

**Grounding**: QLCoder (10.48550/arxiv.2511.08462), VulInstruct (10.48550/arxiv.2511.04014), PATCHEVAL (10.48550/arxiv.2511.11019)

**Implementation**: New ReAct tool `synthesize_targeted_rule(vuln_hypothesis, code_snippet, cwe, language)`. Uses a fast secondary LLM inference step to generate Semgrep YAML, validates against Semgrep schema, runs against the full file. Redis cache with 1-hour TTL. VulInstruct grounds hypothesis in CWE spec language for structured synthesis input.

### 8. Tool-Stall-Adaptive KV Tiering
**Lens**: Efficiency/Performance | **Impact**: HIGH | **Innovation**: MED | **Difficulty**: MED

**Core Idea**: Tokencake says offload KV during tool stalls (47%+ latency reduction). Continuum predicts tool-call duration for TTL assignment. Stateful KV + RoPE warns that non-contiguous eviction breaks positional encoding. Combine: predict stall duration → tiered offload decision (GPU-retain / CPU-pin / disk-evict) → enforce contiguous block eviction only at tail regions to preserve RoPE coherence.

**Grounding**: Tokencake (10.48550/arxiv.2510.18586), Continuum (10.48550/arxiv.2511.02230), Stateful KV + RoPE (10.48550/arxiv.2511.04686)

**Implementation**: `ToolLatencyOracle` with EWM of observed per-tool latencies (seeded with GitHub API p50 priors). Before each tool call, emit Claude `cache_control: {type: "ephemeral"}` checkpoint at boundary (CPU-pin tier) or full prefix checkpoint (disk-evict tier). Short-stall tools (<300ms) skip eviction entirely. RoPE safety: Claude's `cache_control` only evicts the suffix of the cached prefix, never punching holes in the middle.

### 9. Evolutionary CI Adversarial Harness (ECAH)
**Lens**: Security/Reliability | **Impact**: HIGH | **Innovation**: HIGH | **Difficulty**: HIGH

**Core Idea**: ForgeDAN's evolutionary dual-dimensional judgment shows jailbreaks iterate and improve against static defenses. RedCodeAgent found concrete risks in Cursor/Codeium via automated red-teaming. BERT jailbreak detector's reflexivity signal detects novel attacks. Build a `tests/adversarial/` CI suite that evolves its attack corpus on each run using ForgeDAN-style mutation, saving failed defenses as new seeds. Track model reflexivity signals over time as early-warning indicators.

**Grounding**: ForgeDAN (10.48550/arxiv.2511.13548), RedCodeAgent (10.48550/arxiv.2510.02609), BERT jailbreak detector (10.48550/arxiv.2510.01644)

**Implementation**: `tests/adversarial/` with seed corpus in `fixtures/adversarial_diffs/` (three attack categories: embedded instruction injection, NatGVD natural adversarial mutations, BPA suppression phrases near secrets). Each CI run mutates random subset of seeds. Failed defenses → saved to corpus as new seeds. Corpus grows harder over time.

### 10. Asymmetric Phase-Budget Compute for ReAct Observe/Think/Act
**Lens**: Efficiency/Performance | **Impact**: MED-HIGH | **Innovation**: MED | **Difficulty**: MED

**Core Idea**: GateSkip saves 15-50% compute by skipping layers for extraction-only tokens. LiteStage shows per-stage layer budgets beat uniform budgets. ETD concentrates compute in critical middle layers for +28-36% reasoning gains. The ReAct loop has three computationally distinct phases: Observe (extraction-dominant, reasoning-light → 0 thinking tokens), Think (reasoning-critical → 8000 thinking tokens), Act (structured output → 500 thinking tokens). Allocating uniform compute budget to all three wastes inference.

**Grounding**: GateSkip (10.48550/arxiv.2510.13876), LiteStage (10.48550/arxiv.2510.14211), ETD (10.48550/arxiv.2510.07358)

**Expected Gain**: 1.70x speedup + 28-36% reasoning quality improvement on Think phase + ~25% total token cost reduction.

**Implementation**: `PHASE_BUDGETS` dict in `apps/api/src/app/agent/phase_budget.py`. Restructure ReAct loop from single `llm_call()` to explicit `claude_call(phase="observe"|"think"|"act")` with phase-appropriate `thinking_tokens` and `temperature`. Observe: `thinking_tokens=0, temperature=0.0`. Think: `thinking_tokens=8000, temperature=0.1`. Act: `thinking_tokens=500, temperature=0.0`.

### 11. Diff-Type Contextual Evaluation Specification (DCES)
**Lens**: Eval/Quality | **Impact**: MED-HIGH | **Innovation**: MED | **Difficulty**: MED

**Core Idea**: MMTutorBench shows problem-specific rubrics yield significantly higher judge reliability. Omni-Reward shows free-form specs outperform binary preferences. Causal PSC shows prompt design is a decisive factor for code quality assessment. Before running the judge, classify the PR diff into a semantic type (security fix, refactor, new feature, dependency bump, test change) and inject a free-form evaluation specification tailored to that type. Generic rubrics applied to heterogeneous diff types are a primary source of miscalibration.

**Grounding**: MMTutorBench (10.48550/arxiv.2510.23477), Omni-Reward (10.48550/arxiv.2510.23451), Causal PSC (10.48550/arxiv.2511.15817)

**Implementation**: Fast diff-type classifier (small model call) in ARQ job pre-review. Templated `SPEC_TEMPLATES` dict keyed by diff type in `apps/api/src/app/agent/diff_spec.py` (not LLM-generated, avoiding injection surface). Inject spec as prefix to judge system prompt, replacing generic evaluation criteria block. Track `finding_acceptance_rate` segmented by `diff_type`.

## Meta-Architecture Insights (2026-05-06)

### From the Multi-Agent Discussion

**Security/Reliability agent insight**: The four security recommendations form a defensive chain addressing different attack phases — SCD (pre-execution: tool return sanitization), ODQS (execution: targeted rule synthesis), AoFA (post-execution: completeness attestation), ECAH (CI: evolutionary regression). An attacker who defeats one must still defeat the others.

**Efficiency/Performance agent insight**: The four efficiency recommendations have strong systemic synergies:
- REC 1 (KV Tiering) + REC 3 (Sherlock Chains): Speculative tool results arrive just as KV is being repinned from CPU, eliminating re-prefill cost entirely.
- REC 2 (Semantic Tiers) + REC 4 (Phase Budgets): The Observe phase is precisely where noise-tier blocks are identified and evicted — separating phases makes eviction trigger explicit and cheap.

**Eval/Quality agent insight**: The ATLAS (rubric discrimination) + FermiEval (overconfidence) + Multi-Crit (simultaneous criteria failure) interaction reveals a **compound failure mode**: if the rubric contains negatively-discriminating criteria AND the judge is overconfident AND evaluating all criteria in one pass — the judge confidently optimizes for wrong-direction criteria. This is not addressed by any previously proposed item. Solution ordering: (1) Rubric hygiene first (Rec 2), (2) Anchor before judge (Rec 5), (3) Spec before score (Rec 11).

### Emerging Paper-Level Insight for Future Research
The intersection of IRT-based rubric quality assessment + calibrated conformal prediction + diff-type specification suggests a framework for **self-improving evaluation infrastructure** that we have not seen proposed in the literature: an eval system that continuously detects and repairs its own rubric defects using the same signals it generates. This could be the basis of a novel contribution.
