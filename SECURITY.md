# Security Policy

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Report vulnerabilities privately to the maintainers with:

- affected component and version/commit
- reproduction steps
- expected vs actual behavior
- impact assessment

Include a minimal proof of concept when possible. If secrets are involved, redact sensitive values before sharing.

## Coordinated Disclosure

- We acknowledge reports as quickly as possible.
- We validate impact and scope before publishing details.
- We release fixes before public disclosure whenever feasible.

## Scope

This policy covers:

- webhook verification and signature handling
- authentication/session security
- token handling and encryption
- tenant data isolation
- prompt/tool injection defenses
